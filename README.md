## 📘 Project Overview

This project, **ForensicsDetective**, was developed as part of *Assignment 3: Forensic Document Source Classification* under the Bachelor of Artificial Intelligence coursework.  
It focuses on identifying the **origin of PDF documents** by analyzing their visual and structural artifacts through a machine learning–based classification system.  

The primary objective was to train models capable of distinguishing between PDFs generated by four distinct document creation sources:

1. **Microsoft Word**
2. **Google Docs**
3. **Python (ReportLab)**
4. **LibreOffice (FPDF – Fourth Source)**

Each source imprints unique signatures in its exported PDF — including rendering patterns, compression artifacts, and layout metadata — which serve as forensic cues for classification.

## 🧩 Classifiers Implemented

| **Category** | **Model** | **Purpose / Justification** |
|---------------|------------|------------------------------|
| Margin-Based | **Support Vector Machine (SVM)** | Primary classifier as per assignment specification; strong linear decision boundaries |
| Optimization-Based | **SGD Classifier** | Approximates SVM efficiently using stochastic gradient descent; faster convergence |
| Ensemble Method | **Random Forest** | Captures non-linear relationships through decision tree aggregation |
| Neural Network | **MLP Classifier** | Models complex feature interactions for non-linear separation |

All models were trained within a unified pipeline using standardized grayscale image vectors and consistent preprocessing.

---

## 🧪 Methodology and Workflow

| **Stage** | **Description** |
|------------|-----------------|
| **1. Dataset Generation** | Created 10,000 PDFs per source, with randomized text, fonts, and layouts. LibreOffice was chosen as the fourth source for its distinctive export metadata and pixel compression behavior. |
| **2. Image Conversion** | Each PDF was converted to `.png` format using Pillow, enabling pixel-level comparison across sources. |
| **3. Feature Extraction** | All images were resized to **200×200**, converted to grayscale, and flattened into numerical feature vectors. |
| **4. Data Standardization** | Features were scaled using `StandardScaler` for consistent model input. |
| **5. Model Training** | SVM, SGD, Random Forest, and MLP were trained using an **80/20 split** with stratified sampling. |
| **6. Evaluation** | Models were compared via accuracy, F1-score, and confusion matrices. |
| **7. Statistical Validation** | Applied **McNemar’s test** to measure pairwise model significance (p < 0.05). |

## 🧩 Why LibreOffice as the Fourth Source?

LibreOffice was selected as the **fourth document generator** because:
- It is an **open-source, widely adopted office suite** that uses a distinct PDF export engine.
- Its PDFs show **unique font embedding, layout spacing, and compression signatures** distinct from Word and Google Docs.
- These differences ensure **increased dataset diversity** and enhance model generalization across real-world PDF formats.
